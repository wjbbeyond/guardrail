# Security Policy

Please do not open public issues for exploitable security problems.

Report vulnerabilities by opening a private GitHub security advisory or contacting the repository maintainers directly. Include:

- affected version or commit
- reproduction steps
- expected impact
- suggested mitigation, if known

GuardRail handles sensitive prompts and API keys, so reports involving leakage, audit bypass, budget bypass, or upstream credential misuse are treated as high priority.
